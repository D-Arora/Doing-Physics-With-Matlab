svr19.m  (World Data)    svr19CHINA (China Data)

The Script allows one to model the time evolution of the infection caused by the covid19 virus (novel coronavirus) and compare the numerical predictions against the published data. The model parameters are adjusted to best match the actual data. The model can be updated as new data is available. Using the model, we can make predictions about the number of people who will contact the virus, recover and die in the future.

 

The Script is in a development stage. I make changes to the Script as more data becomes available.

 

There has been a global outbreak of the virus since mid-February. To account for additional epicenters of the virus, I have included a surge factor s into the model. As the virus spreads, more people are susceptible to the virus. For a period of days (surge period), the number of susceptible people is increased in the iteration to compute the populations for susceptible S, infected I, recovered R and dead D individuals.

 

The model input parameters are adjusted in the INPUT section of the Script.

 

% ********************************************************************* 

% INPUTS: Model Parameters

% *********************************************************************

 

% Maximum number of days for model   [150]

  tMax = 200;

% Initial number of infections   [0.004]

  I = zeros(num,1);

  I(1) = 0.004;

% Adjustible model parameters 

% S --> I  [0.35]

  a = 0.34;

% I --> R  [0.03]

  b = 0.036;

% I --> D  [0.0022]

  d = 0.0022;

 

% Surge factor: New epicenters for virus  [0 / 0.25]

  s = 0.15;

 

  tsd1 = [2020 02 20];

  tsd2 = [2020 03 30];

 

% Model: number of time steps

  num = 5000;

 

 

The initial population is set to one and the initial numbers of recoveries and deaths are zero.

The Script responsible for the time development of the populations is

 

       for n = 1:num-1

      dS = -a*S(n)*I(n)*dt;

      dI = -dS;

      S(n+1) = S(n) + dS;

      I(n+1) = I(n) + dI;

      dR = b*I(n)*dt;

      dD = d*I(n)*dt;

      R(n+1) = R(n)+ dR;

      D(n+1) = D(n) + dD;

      I(n+1) = I(n+1) - dR - dD;

      N(n+1) = S(n+1) + I(n+1) + R(n+1);

 

   % Surge factor: new epicentres for spread of virus

     if t(n+1) > ts1 && t(n+1) < ts2

       dS = s*S(n+1)*dt;

       S(n+1) = S(n+1) + dS;

     end

    end

 

 

The values from the model and the actual data for I, R and D are displayed graphically. The model parameters are adjusted manually to give a best fit between the model and data values. The model values for I, R and D are scaled to match the actual COVID19 data. The infected population peak is adjusted to equal the maximum number of infections using the scaling variable f.   

% Scaling I, R and D values at day specified by t_index  

  t_index = 27;

  index = find(t > t_index,1);

  f = infected(t_index)./ I(index)

This article describes a simple numerical model to study the time evolution of the COVID19  virus (novel coronavirus). You can use the model to test how well it can predict the number active infections, recoveries and deaths by comparing the model results with actual data. The source of the data is from the website of worldometer

 

       https://www.worldometers.info/coronavirus/

 

This website gives you access to a large amount of data. The daily numbers for active infections, the total number of individuals that have recovered and the total number of deaths are stored in the array covid19.

 

To load the data, type in the Command Window

         load(covid19.mat)
